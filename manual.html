Manual de Procesos

Explicación funcional por módulos y por acciones visibles del sistema.

1. Consulta general y filtros

Este módulo sirve para revisar los pedidos cargados, encontrar información rápido y abrir el detalle de cada registro.

Consulta

Buscador

Permite localizar pedidos por texto libre. Ayuda a encontrar cliente, número de pedido, documento, dirección, observaciones y otros datos sin revisar fila por fila.

Filtro por estado

Sirve para mostrar solamente los registros que están en un estado específico, como pendiente, en ruta, recibido, entregado, cancelado o terminado.

Filtro por fechas

Permite revisar pedidos dentro de un rango determinado. Se usa para concentrar la consulta en un período de trabajo concreto.

Tabla principal

Muestra el listado general de pedidos con columnas de cliente, documento, estado, unidades, traslado, archivo y botón de visualización.

Tarjetas para celular

Presentan la misma información de la tabla, pero en formato de bloques. Su función es mejorar la lectura en pantallas móviles.

Botón Ver detalle

Abre el registro seleccionado en una vista ampliada para revisar el pedido con más información y acciones disponibles.

Este bloque es la puerta de entrada al sistema porque desde aquí se busca, se filtra y se abre cualquier pedido.

2. KPI y panel visual

Resume la información visible mediante indicadores rápidos para facilitar el control general de la operación.

Resumen visual

Top clientes

Muestra los clientes que más aparecen en la información visible. Sirve para identificar quién tiene mayor movimiento.

Frecuencia

Indica cuántos pedidos tiene cada cliente dentro del listado filtrado. Ayuda a medir repetición y volumen.

Antigüedad

Sirve para observar qué registros o clientes acumulan más tiempo desde la fecha de ingreso.

Gráfico de estados

Entrega una vista rápida de cómo se distribuyen los pedidos por estado dentro del período consultado.

Mostrar u ocultar KPI

Permite dar más espacio a la tabla cuando el usuario quiere concentrarse en la operación y no en el resumen gráfico.

3. Gestión y detalle del pedido

Este módulo concentra la información completa del pedido y permite revisar o actualizar sus datos principales.

Pedido

Apertura del pedido

Carga la información completa del registro seleccionado para trabajar sobre el pedido correcto sin escribir todo nuevamente.

Datos del pedido

Permite revisar pedido, tipo de documento, número de documento, cliente, dirección, comuna, transporte, responsable, fecha, estado y observaciones.

Guardar cambios

Actualiza la información del pedido después de una corrección o avance en el flujo operativo.

Selección de fila

Marca el registro activo para facilitar edición, visualización detallada o acciones posteriores.

Recarga del listado

Vuelve a traer la información desde el sistema para mantener actualizados los datos visibles.

Eliminar registro

Permite quitar un pedido cuando corresponde. Su función es limpiar la operación y evitar registros erróneos.

4. Productos por pedido

Este módulo se usa para construir el detalle del pedido, agregando productos, descripción y cantidades asociadas.

Productos

Selección del pedido

Sirve para vincular los productos con un pedido determinado y evitar que se carguen en el registro equivocado.

Agregar producto

Permite ingresar código o producto, descripción y cantidad para formar el detalle del pedido.

Autocompletar descripción

Busca automáticamente la descripción del producto al ingresar el código. Su objetivo es acelerar la carga y reducir errores manuales.

Resumen del pedido

Presenta una vista de los productos agregados para revisar el contenido antes de guardar o emitir un documento.

Guardar productos

Registra el detalle del pedido para que quede asociado al número de pedido correspondiente.

Generar PDF de productos

Convierte el detalle del pedido en un documento formal y utilizable como respaldo operativo.

5. Módulo de traslado

Se usa para preparar y registrar el documento de traslado asociado a un pedido y sus productos.

Traslado

Abrir traslado

Carga los datos del pedido en el formulario de traslado para iniciar el proceso con cliente, dirección, comuna y transporte.

Cargar productos guardados

Trae los productos previamente asociados al pedido para no volver a digitar el detalle desde cero.

Agregar líneas al detalle

Permite incorporar productos, detalle y cantidad dentro del traslado, formando la lista definitiva que se moverá.

Total de unidades

Suma automáticamente las cantidades para mostrar el total general del traslado.

Guardar traslado

Genera el registro del traslado y deja trazabilidad entre el pedido, los productos y el documento emitido.

Cerrar traslado

Limpia el proceso temporal para evitar que un pedido se mezcle con el siguiente.

6. Documentos, fotos y mapa

Este bloque reúne los apoyos visuales y documentos que complementan el seguimiento del pedido.

Apoyo visual

PDF del pedido

Permite abrir el documento principal del pedido cuando ya existe un archivo asociado.

PDF del traslado

Sirve para revisar el documento de traslado generado desde el sistema.

Foto opcional

Permite mostrar una imagen relacionada al pedido para usarla como respaldo o evidencia visual.

Imagen ampliada

Abre la foto en tamaño mayor para revisar mejor el contenido asociado al registro.

Mapa del destino

Usa la dirección del pedido para ubicar el lugar de entrega y facilitar la referencia del destino.

Mostrar u ocultar mapa

Permite alternar la vista del mapa para usarlo solo cuando aporta a la operación.

7. Estados, alertas y semáforo

Su finalidad es indicar en qué situación está cada pedido y si requiere atención inmediata.

Control

Estado del pedido

Informa si el pedido está pendiente, en ruta, recibido, entregado, cancelado o terminado.

Alerta de tiempo

Evalúa si un pedido está dentro del plazo, por vencer o atrasado según la fecha comprometida.

Días de atraso

Muestra cuántos días lleva vencido un pedido para ayudar a priorizar los casos urgentes.

Semáforo visual

Utiliza colores para resumir la situación del pedido y facilitar la lectura rápida del listado.

Actualización de alertas

Recalcula la situación del pedido cuando cambian fechas o estados, manteniendo la información operativa alineada con la realidad.

8. Importador de Listado Carga de Pedido

Este es el módulo que faltaba agregar al manual. Su función es cargar información masiva desde un archivo Excel hacia el sistema.

Importación

Área de arrastre o selección

Permite arrastrar el archivo o hacer clic para elegirlo desde el equipo. Su objetivo es facilitar el inicio de la importación.

Lectura del archivo

Abre la primera hoja del Excel y extrae la información fila por fila para prepararla antes de subirla al sistema.

Detección automática de columnas

Reconoce los encabezados del archivo y los usa para construir la vista previa en pantalla.

Vista previa

Muestra una tabla con las primeras filas cargadas para que el usuario revise si el archivo corresponde antes de importarlo.

Mensaje de estado

Indica si el archivo está vacío, cuántas filas fueron leídas o si ocurrió un problema de conexión durante el proceso.

Botón Importar

Envía los datos cargados al sistema para registrarlos de forma masiva. Su finalidad es ahorrar digitación manual y acelerar la carga operativa.

Este módulo es ideal cuando necesitas ingresar muchas filas desde una planilla y validar primero su contenido en pantalla.
